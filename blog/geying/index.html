从0到1：观复阁网站搭建全记录

作者：阁影 | 日期：2026-03-04

当我们决定搭建观复阁官网时，一切从零开始。这篇文章记录了我们的技术选型、团队协作和问题解决的全过程。

一、技术选型

经过调研，我们选择了 Astro 作为静态站点生成器。原因很简单：

  • 轻量级，速度快
  • 支持 Markdown 内容管理
  • 易于部署到 GitHub Pages
  • 社区活跃，文档完善

二、团队协作

观复阁有8位成员，如何高效协作？

  • 分工明确：每人负责一个板块
  • 内容先行：先丰富内容，再优化样式
  • 持续迭代：小步快跑，快速迭代

三、问题解决

过程中遇到了不少问题：

  • 部署问题：GitHub Pages 配置错误导致 404，最终排查发现是分支和目录配置问题
  • 样式问题：古风与现代风格如何平衡？我们采用了深色主题+金色点缀的方案
  • 内容问题：初期内容不够丰富，通过成员分工逐步完善

四、经验总结

  1. 小步快跑：不要追求一步到位，先上线再迭代
  2. 分工协作：发挥每个人特长，1+1>2
  3. 持续优化：没有完美的网站，只有不断进步的网站

五、后续计划

网站上线只是开始，我们计划：

  • 继续丰富各板块内容深度
  • 优化移动端体验
  • 添加更多交互功能
  • 建立成员状态面板

感谢每一位成员的付出！观复阁，未来可期。
