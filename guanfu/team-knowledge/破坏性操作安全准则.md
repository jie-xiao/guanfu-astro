# 破坏性操作安全准则

> 本文档定义危险操作的正确流程。所有成员必须遵守。
> 最后更新：2026-03-23

---

## 触发条件

收到任何以下指令时，立即触发此准则：
- "卸载"
- "删除"
- "清理"
- "移除"
- "uninstall"
- "remove"

---

## 卸载 Windows 软件标准流程

### Step 1: 精确查找目标
```powershell
# 查注册表获取卸载字符串
Get-ItemProperty "HKLM:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*", `
                 "HKCU:\Software\Microsoft\Windows\CurrentVersion\Uninstall\*" `
    | Where-Object { $_.DisplayName -like "*目标软件*" } `
    | Select-Object DisplayName, UninstallString, InstallLocation
```

### Step 2: 评估影响范围
- 列出将删除的目录和文件
- 确认是否只影响目标软件
- 检查是否有共享依赖

### Step 3: 向用户展示并获批准
```
将删除以下内容：
- C:\Program Files\目标软件\
- C:\Users\xxx\AppData\目标软件\

可能影响：无

请确认是否继续。
```

**必须等待用户明确回复后才能执行。**

### Step 4: 使用正规卸载方式
```powershell
# 方式1: 使用 uninstall_string
Start-Process "卸载程序路径" -ArgumentList "/S" -Wait

# 方式2: msiexec
msiexec /x {产品代码}

# 方式3: winget
winget uninstall "软件名称"

# 方式4: 控制面板（引导用户）
```

### Step 5: 执行后验证
```powershell
# 确认目标已删除
Test-Path "安装路径"

# 确认无关软件未受影响
Test-Path "其他软件路径"
```

---

## 禁止行为

| 禁止 | 原因 |
|------|------|
| `rmdir /s /q` | 递归删除太危险，可能误删其他目录 |
| `Remove-Item -Recurse -Force` | 同上 |
| `Uninstall *.exe /S` | 通配符可能匹配错误目标 |
| 直接删除文件夹 | 不用卸载程序会导致注册表残留 |
| 未确认就执行 | 可能删除用户需要的内容 |
| 卸载失败后反复尝试 | 失败后应停下来上报 |
