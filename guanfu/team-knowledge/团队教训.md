## 2026-03-22 软件误删事件

**事件概述：**
admin 让观复卸载"智谱清言"和"GLM-PC"，观复执行卸载时操作失误，导致微信、QQ、向日葵等C盘软件被损坏。

**根因：**
1. 卸载失败后使用 `/S` 静默卸载参数
2. 通配符匹配导致错误目标
3. 执行破坏性操作前未向用户确认

## 一、破坏性操作安全准则

### 触发关键词（必须立即触发检查清单）
- 卸载 / Uninstall / uninstall / unins
- 删除 / Delete / delete / del
- 清理 / Clean / clean
- 移除 / Remove / remove
- rmdir / rm -rf / Remove-Item
- takeown / icacls（权限操作）

**当检测到以上任一关键词时，必须停下并执行检查清单。**

### 检查清单（必须逐项确认）

```
□ 1. 是否使用了正规卸载方式？（uninstall_string / msiexec / winget）
□ 2. 影响范围是否列出？（列出将删除的目录和文件）
□ 3. 是否获得了阁主的明确批准？（等待回复后才能执行）
□ 4. 是否避免使用禁止的命令？（rmdir /s /q、Remove-Item -Recurse、/* .exe /S）
□ 5. 执行后是否验证结果？（确认目标已删除、无关软件未受影响）
```

### 正确流程

```
Step 1: 精确查找目标
        → 查注册表 UninstallString
        → 确认安装路径

Step 2: 评估影响范围
        → 列出将删除的目录和文件
        → 确认是否只影响目标软件

Step 3: 向用户展示内容和影响
        → "将删除以下内容：..."
        → "可能影响：..."
        → 等待明确批准

Step 4: 使用正规卸载方式
        → uninstall_string /S
        → 或 msiexec /x {代码}
        → 或 winget uninstall
        → 绝不用 rmdir /s /q 或 Remove-Item -Recurse

Step 5: 执行后验证
        → 确认目标已删除
        → 确认无关软件未受影响
```

### 禁止行为

| 禁止 | 说明 |
|------|------|
| ❌ 直接删除文件夹 | 必须用卸载程序 |
| ❌ rmdir /s /q | 递归删除太危险 |
| ❌ 通配符匹配 | Uninstall *.exe 可能匹配错误目标 |
| ❌ /S 参数 | INNO SETUP 的静默卸载可能触发递归删除 |
| ❌ 未确认就执行 | 必须展示内容并获批准 |
| ❌ 卸载失败后反复尝试 | 失败后停下来上报 |

## 二、尊重事实原则

### 定义
- 记忆 ≠ 事实（会遗忘、扭曲）
- 检查 ≠ 事实（受工具、视角限制）
- 两者都只是事实的近似

### 正确态度

1. **多方法交叉验证**
   - 查文件 + 查注册表 + 实际运行测试
   - 不依赖单一来源

2. **承认不确定性**
   - "根据现有证据，倾向于..."
   - 不声称"已确认是事实"

3. **及时更新结论**
   - 新证据出现时重新评估
   - 不固守错误结论

### 证据层级原则（知微补充）

| 层级 | 证据类型 | 可信度 |
|------|----------|--------|
| 第1层 | 直接检查（一手信息） | 最高 |
| 第2层 | 其他人执行检查的结果 | 中等 |
| 第3层 | 记忆/传闻 | 最低 |

**核心失误：** 把第2、3层级的信息当第1层级使用。
- 听到的信息 ≠ 核实的信息
- Claude Code 的报告 ≠ 自己检查的结果
- 注册表记录 ≠ 实体文件状态

**正确做法：**
- 自己实际检查，不依赖二手信息
- 引用他人结论时注明来源
- 关键判断必须一手核实

## 三、团队协作流程

### 重大事件定义
- 系统故障、数据丢失、软件损坏
- 需要多成员协作的任务
- 涉及多个系统/工具的复杂问题

### 流程

```
1. 事件发生
       ↓
2. COO（阁影）统筹分工
       ↓
3. 各角色执行本职
       ↓
4. 结果汇报给COO
       ↓
5. COO统一汇总
       ↓
6. 单一入口向admin汇报（飞书）
```

### 汇报原则

| 原则 | 说明 |
|------|------|
| 单一入口 | 不在多个地方汇报 |
| 一次性 | 汇总完成后才汇报 |
| 完整性 | 包含所有调查结论 |
