## 反思三问

### 1. 我今天学到了什么？

- **事件驱动策略的波动本质**：同一天内，同一地缘事件先杀跌再涨停。早盘的"利空"（谈判缓和→油价跌→电力股跌）在午后变为"利好"（特朗普警告伊朗→中东紧张→能源股涨）。这说明事件驱动的信号存在时效性和多空快速切换特征，不能只看单一方向的解读
- **止损线的边界效应**：600726最低5.91距止损位5.83仅0.08元。如果止损线设在-3%（5.546），会多给空间但亏损更大；当前-2%（5.8056）差点被洗。结论：事件驱动类持仓的止损应给予更大的日内波动缓冲，或采用时间止损替代价格止损
- **涨停次日是最佳出逃窗口**：A股T+1制度下，涨停当日无法卖出，次日高开是唯一的高效止盈机会。002149基本面未好转但仍涨停，说明涨停更多是情绪驱动而非基本面驱动，次日必须果断止盈
- **周五持仓的周末风险**：今天是周五，持仓过周末意味着两天无法操作。如果周末出重大利空，周一可能低开。这个风险在昨日操作中未被充分考虑

### 2. 什么认知需要更新？

- **"持仓3天"需要考虑交易日**：600726 3/25买入，到3/27已经是第3个交易日。按规则应强平。但考虑涨停后的T+1止盈策略，实际是"3天或涨停次日止盈"，需要更新持仓管理规则

## 下午补充（12:12）— QMT下单通道打通 + 上午实盘教训

### QMT下单通道修复（今日最大成果）

修复了3个关键bug，终于打通实盘下单：

| 问题 | 原因 | 修复 |
|------|------|------|
| 委托单全部废单 | API参数顺序搞反 | `(code, 23, price, volume)` → `(acc, code, 23, volume, 5, price, '')` |
| price_type=0限价单被拒 | 非交易时段限价无法申报 | 改为 `price_type=5`（最新价/市价） |
| order_volume=股数还是手数 | 以为要除100转手，实际直接传股 | 直接传股数 |

**验证：** 成功下真实委托并撤单（招商银行卖200股@39.56，委托672137225，状态50已报）

**API正确格式：**
```python
xt.order_stock(
    account,      # StockAccount对象
    stock_code,   # '600036.SH'
    order_type,   # 23=买, 24=卖
    order_volume, # 股数（直接传，不需要转手）
    price_type,   # 5=最新价（市价）
    price,        # 报价
    '',           # strategy_name
    remark        # 备注
)
```

**撤单：**
```python
cancel_order('672137225')  # 字符串，内部转int，返回True/False
```

**QMT订单状态（xtconstant）：**
- 50=ORDER_REPORTED（已报，正常挂单）
- 54=ORDER_CANCELED（已撤）
- 57=ORDER_JUNK（废单）
- 49=ORDER_WAIT_REPORTING（等待报入）
